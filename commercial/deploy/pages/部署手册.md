# Cloudflare Pages 部署手册

## 当前定位

当前商业版激活闭环的主线是：

- Cloudflare Pages
- KV: `LICENSE_KV`
- 双 token：
  - `CLIENT_TOKEN` 给买家端 `cloud` 包
  - `ADMIN_TOKEN` 给卖家本地脚本

当前公开激活地址：

- `https://english-coach-license.pages.dev`

当前已验证通过：

- `/register`
- `/activate`
- `/verify`
- `/v1/chat/completions`
- `/inspect`
- `/revoke`

- Pages project: `english-coach-license`

在 `production` 环境至少保留：

- `CLIENT_TOKEN`
- `ADMIN_TOKEN`
- `CLOUD_API_KEY`
- `LICENSE_SECRET`
- `WORKER_SECRET`

- `SESSION_TOKEN_SECRET`
- `ALLOWED_MODELS`
- `DAILY_REQUEST_LIMIT`
- `DAILY_INPUT_CHARS_LIMIT`
- `BURST_REQUEST_LIMIT`
- `BURST_WINDOW_SECONDS`
- `MAX_PROXY_MAX_TOKENS`

## 日常部署

```bash
cd c:/universe/English/english-coach/commercial/deploy/pages
npx wrangler pages deploy . --project-name english-coach-license --branch main
```

如果本机 `wrangler login` 不稳定，也可以直接使用 `CLOUDFLARE_API_TOKEN`。

## 轮换买家端激活 token

1. 在 Cloudflare Pages 项目里更新 `CLIENT_TOKEN`
2. 本地把同一个值写入构建环境：

```bash
set EC_WORKER_URL=https://english-coach-license.pages.dev
set EC_WORKER_CLIENT_TOKEN=<new-client-token>
python build_cloud.py
```

3. 运行：

```bash
python scripts/check_cloud_activation.py
```

4. 再运行最终商业包 smoke：

```bash
python scripts/smoke_test_release.py --expected-version-mode cloud --portable-exe releases/english-coach-cloud.exe --installer-exe releases/english-coach-cloud-setup.exe --keep-temp
```

## 卖家侧发 key

本地配置文件：

- `private_commercial/seller_cloud_config.json`

当前结构：

```json
{
  "admin_mode": "kv_direct",
  "admin_url": "https://english-coach-license.pages.dev",
  "admin_token": "<admin-token>"
}
```

发 key：

```bash
python send_key.py
```

检查 key：

```bash
python license_keygen.py --inspect XXXX-XXXX-XXXX-XXXX
```

撤销 key：

```bash
python license_keygen.py --revoke XXXX-XXXX-XXXX-XXXX
```

## 成本口径

- Cloudflare Pages + KV 部署成本：可维持在免费层
- 激活/校验/代理后端托管成本：0 现金成本
- 上游 AI 成本：仍取决于 `CLOUD_API_KEY` 对应的真实模型调用量

## 当前建议

- 继续以 `pages.dev` 为主
- `workers.dev` 只保留为历史兼容，不再作为主要发布口径
