# English Coach 操作速查

这份文档只回答一件事：你以后日常到底怎么操作。

## 0. 默认最简单方式

默认直接双击：

- `操作中心.pyw`

它已经内置这些按钮：

- 生成发货文案
- 查 Key
- 撤销 Key
- 开源版完整发布验证
- 商业版完整发布验证
- 仅检查商业激活
- 重新部署商业后端

如果你不想记命令，优先用它。

## 1. 我只想发开源版

在项目根目录运行：

```bash
python build_opensource.py
python scripts/smoke_test_release.py --keep-temp
```

发布物在：

- `releases/english-coach-opensource.exe`
- `releases/english-coach-opensource-setup.exe`
- `releases/english-coach-v2.0.0-opensource.zip`

## 2. 我只想发商业版

先确认私有配置在：

- `private_commercial/cloud_activation_config.json`

然后在项目根目录运行：

```bash
python scripts/check_cloud_activation.py
python build_cloud.py
python scripts/smoke_test_release.py --expected-version-mode cloud --portable-exe releases/english-coach-cloud.exe --installer-exe releases/english-coach-cloud-setup.exe --keep-temp
```

发布物在：

- `releases/english-coach-cloud.exe`
- `releases/english-coach-cloud-setup.exe`
- `releases/english-coach-v2.0.0-cloud.zip`

## 3. 我要给用户发激活码

最简单：

```bash
python send_key.py
```

它会让你选套餐，然后直接生成可发给用户的内容。

如果你要查 key：

```bash
python license_keygen.py --inspect XXXX-XXXX-XXXX-XXXX
```

如果你要撤销 key：

```bash
python license_keygen.py --revoke XXXX-XXXX-XXXX-XXXX
```

## 4. 商业相关文件到底在哪里

公开可维护的商业文件：

- `commercial/seller/`
- `commercial/deploy/pages/`
- `commercial/examples/`

真实私有配置：

- `private_commercial/cloud_activation_config.json`
- `private_commercial/seller_cloud_config.json`
- `private_commercial/key_log.jsonl`

## 5. 根目录里这几个文件为什么还在

下面这些是为了方便你直接操作保留的入口：

- `操作中心.pyw`
- `build_cloud.py`
- `build_opensource.py`
- `send_key.py`
- `license_keygen.py`

它们本身很薄，真正实现已经下沉到别的目录。
